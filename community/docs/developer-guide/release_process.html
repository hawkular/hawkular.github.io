How to release Hawkular

Releasing Hawkular

  • Step 1 - Tagging / building
  • Step 2 - Uploading hawkular and agent to download.jboss.org
  • Step 3 - Update JBoss.org Hawkular Docker images
  • Step 4 - Close JIRA release
  • Step 5 - Update hawkular.org + blog
  • Step 6 - Tweet/G+

Step 1 - Tagging / building

# git clone git@github.com:hawkular/hawkular.git
# cd hawkular
# mvn release:prepare -Prelease
# mvn release:perform

Step 2 - Uploading hawkular and agent to download.jboss.org

1 - Prepare the directories with the files to upload:

1.0.0.Alpha11/
├── hawkular-dist-1.0.0.Alpha11.tar.gz
├── hawkular-dist-1.0.0.Alpha11.tar.gz.md5
├── hawkular-dist-1.0.0.Alpha11.tar.gz.sha1
├── hawkular-dist-1.0.0.Alpha11.zip
├── hawkular-dist-1.0.0.Alpha11.zip.md5
└── hawkular-dist-1.0.0.Alpha11.zip.sha1

2 - Make sure the rights are correct:

chown 444 1.0.0.Alpha11/*

3 - Upload the files

rsync -rv --protocol=28 1.0.0.Alpha11 hawkular@filemgmt.jboss.org:/downloads_htdocs/hawkular/hawkular

Step 3 - Update JBoss.org Hawkular Docker images

1 - Update the DockerFile

# git clone git@github.com:jboss-dockerfiles/hawkular.git
# vi hawkular/hawkular-aio/Dockerfile
// Replace version number, check the WF version number as well
// Test locally "docker build hawkular/hawkular-aio
# git commit -m "Upgrade to Hawkular 1.0.0.Alpha11"
# git push
# git tag 1.0.0.Alpha11
# git push --tags

2 - Build the docker container Go to https://hub.docker.com/r/jboss/hawkular-aio/~/settings/automated-builds/

Step 4 - Close JIRA release

Go to: https://issues.jboss.org/plugins/servlet/project-config/HAWKULAR/versions

Step 5 - Update hawkular.org + blog

# write the blog entry in hawkular.github.io git repo
# update properties.json (this will update the download page)
# update index.html
# Extract release notes from JIRA, take 1.0.0.Alpha11 release as example
# do a PR
# wait for review
# merge

Step 6 - Tweet/G+

Use Hootsuite
